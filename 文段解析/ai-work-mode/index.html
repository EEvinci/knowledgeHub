如何让AI进入疯狂工作模式

2026-05-07 · 微博 · AI Coding
让 AI 进入「疯狂工作模式」意味着让 agent 能够长时间独立运行，不需要人类的持续干预。关键要素：明确的目标声明、状态持久化机制、自动化的验证和重试。

如何让 AI 进入疯狂工作模式

让 agent 连续工作数小时，不需要你盯着。

三个关键要素

1. 明确的目标声明

告诉 agent 要做什么，还要告诉它「做到什么程度才算完成」。没有清晰的成功标准，agent 会一直跑下去，永远觉得自己还没做完。

2. 状态持久化

每隔一段时间，把当前进度写入文件。即使 agent 中断或重启，也能从上次停下的地方继续，而不是从头开始。

3. 自动化的验证和重试

每个子任务完成后自动运行验证。失败了自动重试，而不是停下来等你来看。验证不通过意味着什么？重试几次之后应该放弃？这都需要提前定义。

让 AI 疯狂工作的实际步骤

第一步：把任务拆成 15-30 分钟能完成的小块。太长的话中间容易出问题，太短的话切换开销大。

第二步：为每个子任务定义验证标准。不是「写完代码」，而是「代码能通过测试」「不破坏现有功能」「符合代码风格规范」。

第三步：设置检查点。每完成一个子任务就记录状态，agent 下次启动时先读检查点。

第四步：给 agent 一个「放弃策略」。连续重试 N 次还失败，就记录问题并继续下一个任务，而不是卡死在原地。

一个真实的数字

一个经过良好设计的 agent，可以连续运行 21 小时不需要人工干预。中间遇到问题自动重试，失败了记录日志继续前进。

这不是魔法，是工程。
